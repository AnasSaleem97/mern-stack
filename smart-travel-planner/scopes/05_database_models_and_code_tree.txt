Database Models + Code Tree (MongoDB/Mongoose)

1) Database
- MongoDB database name: smart-travel-planner (default)
- Connection string (recommended local): mongodb://127.0.0.1:27017/smart-travel-planner

2) Mongoose models (backend/src/models)
User
- name, email (unique), contactNumber, password (hashed), role (user/admin), isActive, preferences (Map), createdAt

Destination
- name, country, city, description
- history
- famousLocations[]
- coordinates { lat, lng }
- images[]
- isPopular, isSeasonal, bestSeason, region
- category (enum: Northern Areas, Hill Stations, Valleys, Lakes, Mountains, Beaches, Historical, Cities)
- tagline, culture
- famousFood[], famousThings[], famousFor[]
- suggestedItinerary[]
- weather { current, forecast } (used for attachment; real-time fetch in controller)
- createdAt

Hotel
- (stored in Hotel.js) includes destination ref, coordinates, rating, address, priceRange etc. (see model file for exact fields)

Restaurant
- (stored in Restaurant.js) includes destination ref, coordinates, rating, address etc.

Budget
- stores userId, destination, numberOfMembers, days, season, breakdown totals + recommendations/insights; used by MoneyMap + TravelFund

BudgetRule
- stores rules for calculations and admin-managed rules

ApiKey
- stores external API keys (managed via Admin Panel)

BuddyBotConversation
- stores per-user conversation history for BuddyBot
